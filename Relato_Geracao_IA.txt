3. DESCRIÇÃO DO PROBLEMA

O projeto Wazuh ATT&CK Mapper foi concebido para resolver uma dificuldade comum enfrentada por analistas de SOC (Security Operations Center).

Embora o Wazuh gere alertas de segurança contendo identificadores de regras, descrições e informações de contexto, nem sempre existe uma associação explícita e consolidada com as técnicas e táticas do framework MITRE ATT&CK.

Como consequência, os analistas precisam interpretar manualmente cada alerta para compreender sua relação com comportamentos adversários conhecidos.

O objetivo do artefato foi automatizar esse processo, enriquecendo alertas do Wazuh com informações do framework MITRE ATT&CK e produzindo relatórios estruturados.

4. PROCESSO DE DESENVOLVIMENTO ASSISTIDO POR IA

O desenvolvimento ocorreu de forma incremental.

Inicialmente foi definido o problema de pesquisa e o nome do projeto:

"Wazuh ATT&CK Mapper"

Posteriormente foi criada a estrutura do repositório GitHub contendo os diretórios:

• src
• tests
• examples
• data
• reports

Em seguida foram desenvolvidos os componentes do sistema:

4.1 Parser de Alertas

Foi solicitado à IA a geração do módulo parser.py responsável por:

• Ler arquivos JSON contendo alertas Wazuh
• Extrair Rule ID
• Extrair descrição
• Extrair grupos
• Extrair nome do agente

O código gerado foi testado e refinado após validações realizadas manualmente.

4.2 Base de Mapeamento MITRE

Foi criada uma base de conhecimento local chamada mitre_mapping.json contendo mapeamentos entre Rule IDs do Wazuh e técnicas MITRE ATT&CK.

Foram incluídas regras relacionadas a:

• Brute Force
• Valid Accounts
• Privilege Escalation
• SQL Injection
• Discovery
• Execution

4.3 Mecanismo de Enriquecimento

Foi desenvolvido o módulo mapper.py responsável por:

• Ler o arquivo mitre_mapping.json
• Localizar Rule IDs
• Associar técnica ATT&CK
• Associar tática ATT&CK
• Tratar regras sem mapeamento

4.4 Geração de Relatórios

Foi desenvolvido o módulo report_generator.py responsável por:

• Gerar relatórios CSV
• Criar automaticamente o diretório reports
• Exportar alertas enriquecidos

4.5 Testes Automatizados

Foram desenvolvidos testes utilizando pytest para validar:

• Leitura de alertas
• Estrutura dos dados processados
• Mapeamentos MITRE válidos
• Tratamento de regras desconhecidas
